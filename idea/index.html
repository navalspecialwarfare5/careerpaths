NSW Joint Operations Portal

Current Development Plan for Multi-Unit Dashboard Infrastructure

1. Introduction

The NSW Joint Operations Portal is a secure, centralized web-based environment designed to support realistic inter-unit cooperation in ARMA 3. This document outlines the current development roadmap, system capabilities, and modular features that will be available to approved unit leadership. It is intended as a working plan to guide the build and gather feedback.

2. System Overview

  • Accessible through a secure browser dashboard
  • Role-based access for unit leaders, staff, and operators
  • Stores mission documents, unit data, qualification records
  • Runs entirely in a browser, no external dependencies required

3. Key Features

  • OPORD / AAR / WARNO / INTSUM Generator – static templates with preview, archive, and edit options
  • Document Sorting System – searchable by unit, date, mission type, and AO
  • Custom Unit Landing Page – each unit has a designated view with their own documents and roster
  • LocalStorage-Based Persistence – fully client-side document saving (GitHub Pages-compatible)

4. Unit Management

  • Unit Admin Control – unit reps manage only their own unit
  • Manual Account Creation – unit owners create & manage their members
  • Role Assignment – basic user, staff officer, or administrator per unit
  • Unit Emblem & Colors – cosmetic control over unit branding on the dashboard

5. Personnel Tools

  • Individual roster view with active/inactive tracking
  • View qualifications, assignment history, and known callsigns
  • Strike tracking (optional) for discipline or warnings
  • Unit-internal promotion notes and user notes (visible to staff)

6. Document Infrastructure

  • Preview, edit, archive, and duplicate documents
  • Support for templated briefing documents with timezones & formatted mission data
  • File tagging (e.g., “Active”, “Historical”, “Training”)
  • Optional upload of external briefings or imagery (.jpg, .png, .pdf)

7. Security & Access

  • Basic login system with hashed passwords
  • Role-based permissions: View / Edit / Delete
  • Unit isolation: one unit cannot see another’s documents unless authorized
  • Optional clearance tags (NOFORN, EYES ONLY – visual, not functional)

8. Realistic Limitations

  • No integration with live operations or server-side telemetry
  • No mission deployment automation
  • No persistent cloud database (uses browser localStorage for now)
  • All actions are non-destructive and roleplay-friendly (no backend enforcement)

9. Future Capabilities

  • Google Slides API integration for live in-game briefings
  • Arma 3 TOC laptop overlay system with mission-linked document view
  • In-game monitor synchronization via scripted variable names
  • Firebase or secure backend for long-term document storage (optional)

10. Final Notes

This system is not intended to simulate classified or sensitive real-world operations. It is a fictional, ARMA 3-enhanced planning tool for immersive joint operations. All feedback is welcome during development, and features can be tailored to suit unit needs.
